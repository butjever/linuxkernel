#### 项目题目: Hardware accelerator for eBPF
#### 简要描述
eBPF 是 Linux 内核主线支持的字节码技术，允许在内核态执行非可信用户代码，为网络可观测性提供了必要的基础设施。然而 eBPF 处理网络数据包的性能受限于 OS，且无法介入 Kernel bypass 场景下网络应用的数据通路，难以充分满足高性能网络对可观测性的需求。网络硬件尚不具有通用的可编程规范，使得基于 eBPF 的网络可观测性方案难以下推至既有硬件；而通过在硬件层面实现 eBPF 字节码执行能力，即可复用 Linux 生态下的可观测性方案。本次分享将介绍硬件 eBPF 的现状与我们基于数据流架构实现硬件 eBPF 加速器的工作。
 
 
#### 作者简介
周鹤洋（GitHub: losfair），目前就读于南京航空航天大学，此前在字节跳动、Vercel 等公司实习，现正与 DatenLord 团队合作完成毕业设计项目 WaveBPF 硬件加速器。开源软硬件开发者、独立产品 Planet 作者，发起和参与过 Blueboat、Wasmer、Next.js 等开源项目，业余开发过超标量处理器 Violet、内核态 WebAssembly 运行时 kernel-wasm 等 OS 与硬件项目。

#### 项目题目: Extend Database Mesh with eBPF
#### 简要描述
 在数据库上云的大趋势下，越来越多的企业开始寻找一个能对数据库进行完整治理的有效方案，解决数据库碎片化、SQL 流量治理、数据库可靠性工程等诸多场景中的问题。而 Database Mesh 即对这些问题提供了云原生的解决方案。在实际生产场景中，为了进一步保障业务可靠性，优化资源利用率，需要对不同优先级的业务应用提供不同的数据库流量 QoS，而 eBPF 的出现为此提供了不依赖于硬件的解决方案。本次分享将介绍 Database Mesh 如何利用 eBPF 为业务应用提供云原生的数据库流量 QoS 治理方案。
 
#### 作者简介

武建东，SphereEx Cloud Engineer，深耕云原生、Linux 等技术多年，开源狂热爱好者，对云基础设施自动化、DSL 有浓厚兴趣，目前专注于 Database Mesh 方向，致力于通过各种技术手段提升企业数据库上云体验。
